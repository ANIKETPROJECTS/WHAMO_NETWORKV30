🧩 Feature 1: Node Ascending Order Validation
📌 Requirement
All connected nodes in a pipeline/network path must follow ascending order

Rule:

Current Node > Previous Node

Example:

✅ Valid:   N2 → N3 → N10 → N12 → N15
❌ Invalid: N2 → N3 → N10 → N12 → N1 → N15
                             ↑ ERROR
⚙️ Behavior
While:
Creating connections
Editing node numbers
Loading saved project

Validate sequence:

if (currentNode <= previousNode) {
    // trigger error
}
🚨 UI Feedback
Highlight invalid node:
Red border or glow
Highlight invalid connection path

Show message:

"Node order must be in ascending sequence. Node 1 cannot come after Node 12."
🧠 Validation Logic (Frontend)
function validateNodeSequence(nodes) {
  let errors = [];

  for (let i = 1; i < nodes.length; i++) {
    if (nodes[i] <= nodes[i - 1]) {
      errors.push({
        index: i,
        message: `Node ${nodes[i]} must be greater than Node ${nodes[i - 1]}`
      });
    }
  }

  return errors;
}
