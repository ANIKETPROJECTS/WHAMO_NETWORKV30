## BACKGROUND — What PCHAR TYPE 3 is

PCHAR TYPE 3 is a pump characteristic block used in waterhammer/transient 
simulation INP files. It looks like this:

PCHAR TYPE 3
 SRATIO  -2.0  -1.5  -1.0  -0.5  0.0  0.5  1.0  1.5  2.0
 QRATIO  -2.0  -1.5  -1.0  -0.5  0.0  0.5  1.0  1.5  2.0
 HRATIO
   (one row per SRATIO value, one value per QRATIO column)
 TRATIO
   (one row per SRATIO value, one value per QRATIO column, 8 per line)
FINISH

Where:
- SRATIO = speed ratio = actual speed / rated speed
- QRATIO = flow ratio  = actual flow  / rated flow
- HRATIO = head ratio  = actual head  / rated head
- TRATIO = torque ratio = actual torque / rated torque

This covers ALL FOUR QUADRANTS of pump operation including:
  normal pumping, reverse flow, runaway, and turbine mode.

  angle  t_value  ← pairs of (theta_degrees, WB value)

(repeats for each specific speed: 1280/25, 1800/35, 4850/94, 7500/145, 
13500/260 for CircularFormat, and more for Suter_ChaudhryFormat and 
Unused format)

---

## CURVE FORMATS EXPLAINED

### 1. CircularFormat
Uses the Suter/circular representation:
- X-axis is angle theta in DEGREES (0 to 360)
- theta = atan2(Q_ratio, N_ratio) converted to degrees (in all 4 quadrants)
- WH(theta) = H_ratio / (N_ratio² + Q_ratio²)  → head characteristic
- WB(theta) = T_ratio / (N_ratio² + Q_ratio²)  → torque characteristic

To RECOVER H_ratio and T_ratio from CircularFormat at a point (N_ratio, Q_ratio):
  theta = atan2(Q_ratio, N_ratio) in degrees (0-360)
  denom = N_ratio² + Q_ratio²
  H_ratio = WH(theta) × denom
  T_ratio = WB(theta) × denom

Interpolate WH and WB at required theta using linear interpolation between 
the two nearest data points in the file.

### 2. Suter_ChaudhryFormat
Same as CircularFormat but theta is in RADIANS (0 to 2π).
Conversion: theta_radians = theta_degrees × π/180

### 3. Unused
This format is also in RADIANS but uses a different angular convention. 
SKIP these entries — do not use them for auto-generation.

## ALGORITHM — How to Generate PCHAR

Given user inputs H_rated, Q_rated, N_rated and selected Ns curves:

STEP 1 — Build SRATIO and QRATIO arrays from user grid inputs.
  Example: SRATIO = [-1.5, -1.25, -1.0, ..., 1.25, 1.5]
           QRATIO = [-1.5, -1.25, -1.0, ..., 1.25, 1.5]

STEP 2 — Load the HEAD and TORQUE data for the selected specific speed 
from the predefined file. Parse all (angle, value) pairs.

STEP 3 — For each combination of (s, q) in SRATIO × QRATIO:

  a. Compute theta:
     For CircularFormat:  theta = atan2(q, s) in degrees, mapped to [0, 360)
     For Suter format:    theta = atan2(q, s) in radians, mapped to [0, 2π)

  b. Interpolate WH(theta) and WB(theta) from the loaded curve data using 
     linear interpolation between the nearest two angle values.
     Handle wraparound: if theta is beyond the last point, interpolate 
     between last and first point (circular).

  c. Compute denominator:
     denom = s² + q²
     If denom < 1e-6, set H_ratio = 0 and T_ratio = 0 (avoid divide by zero)
     Else:
       H_ratio = WH(theta) × denom
       T_ratio = WB(theta) × denom

  d. Store H_ratio and T_ratio for this (s, q) pair.

STEP 4 — Format the PCHAR block:

PCHAR TYPE 3
 SRATIO  [space-separated SRATIO values, 2 decimal places]
 QRATIO  [space-separated QRATIO values, 2 decimal places]
 HRATIO
   [one row per SRATIO value]
   [each row has one H_ratio per QRATIO value, 4 decimal places]
 TRATIO
   [one row per SRATIO value]
   [each row has one T_ratio per QRATIO value, 4 decimal places, 8 per line]
FINISH
